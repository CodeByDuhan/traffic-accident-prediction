# Corrected Heavy-Rain Augmentation Report

## Purpose

This report explains the corrected heavy-rain augmentation strategy used in the traffic accident prediction project.

The main objective of this step is not only to increase the number of heavy-rain samples, but to help the model learn that heavy rain is a **context-dependent non-linear signal**.

In the original dataset, precipitation values greater than 5 mm were relatively rare compared to normal weather conditions. Because of this imbalance, early models did not learn heavy-rain behavior reliably.

However, heavy rain should not be interpreted as a simple rule such as:

> More rain always means higher accident risk.

After precipitation becomes high, driver behavior may change. Some drivers may slow down, become more cautious, or avoid traffic completely. Therefore, heavy rain can produce different risk patterns depending on the surrounding weather and time context.

---

## Main Hypothesis

The final augmentation strategy is based on the following hypothesis:

> When precipitation is greater than 5 mm, accident risk becomes context-dependent. Heavy rain can indicate high risk under dangerous conditions, but it should not automatically force the model to predict an accident in every case.

Because of this, the dataset was augmented by separating heavy-rain samples into two groups:

| Context | Meaning |
|---|---|
| Safe heavy-rain context | Heavy rain exists, but surrounding conditions are relatively safe. |
| Risky heavy-rain context | Heavy rain exists together with additional risk-increasing factors. |

---

## Safe Heavy-Rain Context

Safe heavy-rain samples represent cases where precipitation is high, but the surrounding conditions are not strongly dangerous.

These samples help prevent the model from learning a wrong shortcut such as:

> If `precipitation > 5`, always predict accident.

The safe context is based on conditions such as:

```python
precipitation > 5
10 <= temperature_2m <= 25
windspeed_10m < 10
9 <= hour <= 18
accident_happened_t+1 == 0
```

These conditions represent heavy-rain cases where:

- the temperature is not extreme,
- wind speed is not high,
- the time is during daytime,
- and the target label is negative.

This teaches the model that heavy rain can exist without necessarily producing a positive accident-risk label.

---

## Risky Heavy-Rain Context

Risky heavy-rain samples represent cases where precipitation is high and additional risk factors are present.

These samples help the model learn that heavy rain can still be a strong accident-risk signal when combined with dangerous context.

The risky context is based on conditions such as:

```python
precipitation > 5
and (
    is_night == 1
    or high_wind == 1
    or dangerous_temp == 1
)
accident_happened_t+1 == 1
```

These conditions represent cases where heavy rain appears together with one or more of the following:

- night time,
- high wind,
- dangerous temperature.

This teaches the model that heavy rain should be interpreted together with surrounding conditions, not as an isolated feature.

---

## Augmentation Groups

The corrected augmentation process creates additional samples for both safe and risky heavy-rain situations.

| Augmentation Group | Description |
|---|---|
| `safe_5_10_neg` | Safe heavy-rain negative samples where precipitation is between 5 and 10 mm. |
| `safe_gt_10_neg` | Safe heavy-rain negative samples where precipitation is greater than 10 mm. |
| `risky_5_10_pos` | Risky heavy-rain positive samples where precipitation is between 5 and 10 mm. |
| `risky_gt_10_pos` | Risky heavy-rain positive samples where precipitation is greater than 10 mm. |

This structure prevents the augmentation process from only increasing positive heavy-rain examples. Instead, it creates a more balanced heavy-rain representation across both negative and positive contexts.

---

## Numeric Jitter Strategy

Augmented rows are generated by applying small numeric changes to selected weather features.

The jitter is applied to features such as:

- `temperature_2m`
- `windspeed_10m`
- `precipitation`

The target label and context are preserved.

This means the augmented samples remain close to the original data distribution while still giving the model more examples of rare heavy-rain situations.

The purpose of jitter is not to create unrealistic weather cases. Its purpose is to create controlled variations of existing rare samples.

---

## Leakage-Safe Splitting

A critical part of the corrected pipeline is preventing data leakage between train and test sets.

If an original row is placed in the training set and its augmented copy is placed in the test set, the evaluation result becomes unreliable. The model may appear to perform well because it has already seen a nearly identical version of the test sample during training.

To avoid this, each original row receives a `source_id`.

Augmented rows keep the same `source_id` as the original row they were generated from.

Then train/test splitting is performed with `GroupShuffleSplit` using `source_id` as the group key.

This ensures that:

> An original row and all of its augmented versions always stay in the same split.

As a result, there is no source-level overlap between train and test sets.

## Final Dataset Validation

After the corrected augmentation and group-based splitting process, the final dataset statistics were:

| Item | Value |
|---|---:|
| Original rows | 2,130,543 |
| Corrected rows | 2,148,243 |
| Added rows | 17,700 |
| Train rows | 1,718,663 |
| Test rows | 429,580 |
| Train ratio | 0.8000 |
| Test ratio | 0.2000 |
| Source ID overlap between train/test | 0 |

The most important validation result is:

```text
Source ID overlap between train/test = 0
```

This confirms that augmented rows did not leak across train and test sets.

---

## Why This Matters

The corrected heavy-rain augmentation strategy improves the dataset in two ways.

First, it increases the number of rare heavy-rain examples so that the model can observe these cases more often.

Second, it prevents the model from learning an oversimplified relationship between precipitation and accident risk.

Instead of teaching the model:

> Heavy rain always means accident.

The corrected pipeline teaches:

> Heavy rain should be interpreted together with time, wind, temperature, and context.

This is especially important because the final model is expected to learn contextual weather-risk patterns rather than memorize a single threshold rule.
